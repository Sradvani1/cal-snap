# PR8: Settings Screen

**Status:** Implemented  
**Source of truth:** [docs/technical-spec.md](../technical-spec.md) (PR 8), [PR-06.md](./PR-06.md), [PR-07.md](./PR-07.md)

## 1. Objective

Deliver the Settings tab with profile editing/recalculation, partner add/remove, API key management, HealthKit toggles and active-user sync, weigh-in reminder scheduling (PR6 keys), unit preferences, CSV export, and scoped data deletion.

---

## 2. In scope

- **Settings tab** — third `TabView` entry with `NavigationStack`
- **`SettingsView` + `SettingsViewModel`** — all sections inline in one view file
- **`ProfileUpdateService`** — preview/apply recalculation + macro slider logic
- **`DataExportService`** — CSV generation
- **`UserDataDeletionService`** — scoped delete + per-user AppStorage cleanup
- **`AddPartnerFlowView`** — reuses `OnboardingViewModel` + existing step views
- **HealthKit** — write gating, `fetchLatestWeight()`, active-user-only Sync Now
- **`profileDataRevision`** — cross-tab refresh for Dashboard and Analytics
- **Unit tests** — `testMacroSliderValidation`, `testRecalculationOnProfileEdit`, `testCSVExport`

---

## 3. Out of scope

- Daily log reminders (PR10 — approved deferral)
- Design system polish (PR9)
- Schema changes, new SPM packages

## 6. Spec extensions (approved)

1. **Daily log reminder → PR10** — PR8 ships weekly weigh-in reminder UI only
2. **Sync now** — active-user latest HK body mass import only (≥0.1 kg delta)
3. **`profileDataRevision`** — sole cross-tab refresh mechanism
4. **Profile weight save** — creates `WeighIn` via `WeighInService` when weight changes
5. **HK write gating** — AppStorage toggles at call sites; failures log-only
