## 4. Key implementation details

### Daily log reminders

- Extended `NotificationManager` (injected, no singleton) with `scheduleDailyLogReminder` / `cancelDailyLogReminder`
- Settings: toggle (default off) + time picker in Notifications section
- Tap opens meal scanner via `AppNavigationCoordinator` (cold-launch `pendingScannerOpen` parallel to weigh-in sheet)

### Widgets

- Target: `CalSnapWidget` (`com.calsnap.app.widget`)
- App Group: `group.com.calsnap.shared`
- Write points: `DashboardView.reloadDashboard()`, Settings profile save, data deletion clear

### App Intents

- `OpenScannerIntent` + `CalSnapShortcuts` — opens scanner only (no auto-log)

### Localization

- Phase 1 complete for notification copy in `Localizable.xcstrings` (neutral bodies; no interpolated names)
- **Deferred (post-PR10):** phase 2 migration of remaining ~30 feature files — tracked explicitly below; not a merge blocker for widgets/notifications/intents

## 9. Post-review fixes (1–5)

1. **Navigation coordinator** — `AppContainer` owns `AppNavigationCoordinator`; `AppNavigationCoordinatorStore.bind()` wires intents to the same instance SwiftUI observes.
2. **Notification strings** — weigh-in and daily-log copy use neutral `String(localized:)` keys only (no interpolated display names).
3. **Localization deferral** — phase 2 (~30 feature files) completed in PR11; see [PR-11.md](./PR-11.md).
4. **WidgetSyncService test** — `WidgetSyncServiceTests` covers `makeWidgetData` mapping; dashboard sync overload simplified.
5. **Widget build settings** — CalSnapWidget target: iOS 26, Swift 6, strict concurrency.
