### Naming convention

`{feature}.{surface}.{element}` — e.g. `dashboard.greeting.morning`, `common.button.save`, `mealScanner.alert.discardMeal.title`

Shared literals deduplicated under `common.*`, `error.*`, `designSystem.*`, `units.*`.

### Interpolation policy

Manual catalog keys with `%lld` / `%@` placeholders use:

```swift
String(format: String(localized: "dashboard.netCalories.overGoal"), delta)
Text(String(format: String(localized: "designSystem.calorieRing.ofGoal"), target))
```

Bare `String(localized: "key \(arg)")` does not resolve for manual keys.

## 6. Known exceptions (intentional remaining literals)

| Location | Reason |
|----------|--------|
| `GeminiService` prompt strings (`"Reply with OK"`, analysis prompts) | Not user-facing UI |
| `OpenScannerIntent` Siri `phrases` array | App Shortcuts metadata requires `\(.applicationName)` at compile time; title/description/shortTitle are catalog-backed |
| `CameraImagePicker` preview-only message | `#Preview` / device-only |
| Dynamic AI/user content (food names, insights, estimation notes) | `Text(verbatim:)` |
| Chart `.value("Day", …)` axis keys | Internal chart identifiers, not standalone UI copy |
| SF Symbol names, Codable raw values, Logger/debug | Non-UI |
| `#Preview` mock data | Test fixtures |
| `error.localizedDescription` passthrough | System/framework errors |
