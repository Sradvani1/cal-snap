# PR7: Analytics & Insights Screen

**Status:** Implemented  
**Source of truth:** [`docs/technical-spec.md`](../technical-spec.md) → **PR 7: Analytics & Insights Screen**; [`PR-05.md`](./PR-05.md); [`PR-06.md`](./PR-06.md)

---

## 1. Objective

Deliver the Analytics tab with timeframe-driven meal aggregation, Swift Charts sections, embedded weight progress from PR6, and on-demand Gemini insight generation from aggregated stats only.

---

## 2. In scope

- **`AnalyticsDateRange`** + **`AnalyticsAggregator`** — pure aggregation (adherence, macros, fiber, patterns, top foods)
- **`MealRepository.fetchMeals(for:from:through:)`** — range fetch mirroring weigh-in window
- **`AnalyticsView` + `AnalyticsViewModel`** — load pipeline, empty-state gate, insight generation
- **Chart sections** — calorie adherence, macro trends, fiber, patterns (DOW, time-of-day, top 5)
- **`WeightProgressPresentation.embedded`** — minimal `WeightProgressView` refactor for Analytics embed
- **`GeminiService.generateAnalyticsInsight(_:)`** — plain-text prompt from aggregates only
- **`TabView`** in `RootView` — Dashboard + Analytics tabs
- **Unit tests** — `testAdherenceCalculation`, `testDayOfWeekBreakdown`, `testTopFoodsAggregation`

---

## 3. Out of scope

- Settings tab (PR8)
- Alcohol / highest-meal analytics (product-research only)
- Automatic insight scheduling
- Schema changes, new SPM packages

## 6. Spec extensions

1. **Adherence denominator** — `% days on target` = `onTargetLoggedDays / loggedDayCount`. Unlogged days excluded.
2. **Fiber target** — `NutritionCalculator.fiberTargetG(dailyCalorieTarget:)`; supersedes fixed 25g prose in spec.
3. **Weight section timeframe** — All-time weigh-ins via PR6 `WeightProgressViewModel.load()`.
4. **Custom range** — Capped at 365 days; start/end normalized and swapped if inverted.

## 9. Post-review fixes

| Issue | Fix |
|-------|-----|
| First-load race (split `.task` modifiers) | Single `.task(id: reloadToken)` creates VM and reloads |
| Custom timeframe cancel desync | Revert `timeframePreset` on sheet dismiss when range not `.custom` |
| Insight cleared on every reload | Remove clearing from `load()`; clear on range/profile change only |
| Weight VM recreated every reload | Recreate only when profile changes; bump `reloadTrigger` on profile change or weigh-in save |
| Adherence band duplicated | `CalorieProgressBand.progressBand` + `isCalorieIntakeOnTarget` shared by dashboard and analytics |
| API key error copy | Analytics insight shows Settings-specific message for `apiKeyMissing` |
