# PR1: Project Scaffold & Core Infrastructure

**Status:** Implemented  
**Source of truth:** [`docs/technical-spec.md`](../technical-spec.md) (PR 1 section), [`docs/engineering-rules.md`](../engineering-rules.md), [`docs/product-research.md`](../product-research.md) (§12 macro defaults), [`README.md`](../../README.md), [`.cursor/rules/projects.mdc`](../../.cursor/rules/projects.mdc)

## 1. Objective

Establish the CalSnap iOS project skeleton with SwiftData persistence wired at launch, all four core data models, shared constants and enums, and the full `NutritionCalculator` + `KeychainManager` business-logic layer with unit test coverage. The only user-visible UI is a blank `RootView` placeholder; no onboarding, dashboard, external APIs, or third-party packages.

---

## 2. In scope

- New Xcode project (`CalSnap.xcodeproj`) — SwiftUI App lifecycle, SwiftData enabled, iPhone-only, iOS 17.0, bundle ID `com.calsnap.app`
- [`.gitignore`](../../.gitignore) for Xcode/SwiftPM artifacts and Cursor plan artifacts
- App entry: `CalSnapApp.swift` with `modelContainer(for:)` for all four models
- Blank `RootView` (solid background, `#Preview` only)
- Empty `@Observable` `AppContainer` injected via `.environment(appContainer)` — no custom `EnvironmentKey`
- [`Constants.swift`](../../CalSnap/Core/Utilities/Constants.swift) — full `AppConstants` as spec
- [`Enums.swift`](../../CalSnap/Core/Models/Enums.swift) — `BiologicalSex`, `ActivityLevel`, `MealType` + `MealType.suggested(for:)` using technical-spec hour ranges verbatim
- Four SwiftData `@Model` classes with documented initializers
- `UserProfile` macro defaults at creation: protein `0.28`, carbs `0.47`, fat `0.25`
- [`NutritionCalculator.swift`](../../CalSnap/Core/Services/NutritionCalculator.swift) — all static methods; `isOnPlateau` uses nil-safe min/max
- [`KeychainManager.swift`](../../CalSnap/Core/Utilities/KeychainManager.swift) — `KeychainKey`, `KeychainError`, save/load/delete + `String` account overloads for tests
- Unit tests: `NutritionCalculatorTests` (9 cases), `KeychainManagerTests` (1 round-trip case)
- `project.yml` — XcodeGen spec to regenerate `CalSnap.xcodeproj`
- Minimal build artifacts: `Assets.xcassets`, minimal `Info.plist`

## 6. Implementation steps

1. Create Xcode project via `project.yml` + `xcodegen generate` (iOS 17, iPhone only, SwiftData, no SPM)
2. Add `.gitignore` at repo root
3. Create folder groups: `App/`, `Core/Models/`, `Core/Services/`, `Core/Utilities/`, `Resources/`
4. Implement `Constants.swift` from technical-spec § Global Constants
5. Implement `Enums.swift` with `MealType.suggested(for:)` ranges: `5..<11`, `11..<15`, `15..<18`, default `.dinner`
6. Implement SwiftData models with documented inits (`UserProfile` macro defaults `0.28/0.47/0.25`; `WeighIn` computed fields default `0`; date defaults use `?? Date()`)
7. Implement `NutritionCalculator.swift` (nil-safe `isOnPlateau`; `_ = currentTDEE` retained for PR6)
8. Implement `KeychainManager.swift` with `kSecAttrService = "com.calsnap.app"` on all queries (see spec extensions below)
9. Implement `AppContainer.swift` — empty `@Observable`, DI comment, no custom `EnvironmentKey`
10. Implement `RootView.swift` — `Color(.systemBackground)` only
11. Implement `CalSnapApp.swift` — `modelContainer(for: [UserProfile.self, MealEntry.self, FoodItem.self, WeighIn.self])`
12. Add `NutritionCalculatorTests.swift` (9 tests)
13. Add `KeychainManagerTests.swift` (round-trip with `com.calsnap.test_key`)
14. Build & test: `xcodebuild -scheme CalSnap -destination 'platform=iOS Simulator,name=iPhone 16' test`

## 9. Risks and edge cases

| Risk | Mitigation |
|------|------------|
| Keychain test key vs `KeychainKey` enum | `String` account overloads; tests use `com.calsnap.test_key` only |
| `isOnPlateau` spec force-unwraps | Nil-safe `min() ?? 0` / `max() ?? 0` (documented spec extension) |
| `weightProjection` safety floor | Fixed test inputs (80 kg, 350 kcal) verified monotonic offline |
| Keychain test pollution | `setUp`/`tearDown` delete test account; iOS Simulator only |
| `isOnPlateau` assumes sorted weigh-ins | Caller must pass chronological order (PR6) |

### Spec extensions (document in PR description)

1. **`isOnPlateau`** — uses nil-safe min/max instead of force-unwraps.
2. **`KeychainManager`** — adds `kSecAttrService: "com.calsnap.app"` to namespace keys (technical-spec snippet omits this).
3. **`KeychainManager`** — adds `String`-based account overloads for test isolation.
4. **`UserProfile` init** — date defaults use `?? Date()` instead of force-unwrap.
