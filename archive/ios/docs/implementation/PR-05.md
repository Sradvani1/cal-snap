# PR5: Meal Detail, Edit & Daily Log

**Status:** Implemented

## 1. Objective

Complete the daily log experience: view logged meals, edit via pre-populated scanner, delete with SwiftData removal + HealthKit reversal, dashboard navigation (tap / swipe / context menu), and Daily Summary Footer.

---

## 2. In scope

- **`MealHealthSnapshot`** — value-type pre-delete/pre-edit HK capture
- **`MealRepository`** — `fetchMeal`, `delete` (throws `MealRepositoryError.mealNotFound`), `update` (in-place, replace child items)
- **`HealthKitService.reverseMeal(_:)`** — negative dietary samples
- **`MealDeletionService`** — shared snapshot → delete → HK reverse flow
- **`MealScannerViewModel` edit mode** — `loadForEditing`, `saveMeal`, `updateMeal`
- **`Features/MealLog/`** — `MealDetailView`, `MealListView`, `MealShareCardView`, `DashboardRoute`
- **`DailySummaryFooterView`** — fiber, net kcal, macro % text
- **Typed `NavigationStack(path:)`** — `DashboardRoute` / `MealScannerRoute`
- **Non-scrolling embedded `List`** in dashboard `ScrollView` for swipe actions
- **Unit tests** — `testMealDeletion`, `testMealEdit`, repository error cases, footer computed props

---

## 3. Out of scope

## 6. Spec extensions

1. **HealthKit reversal** — `MealHealthSnapshot` + negative samples; no HK UUID fields.
2. **Delete flow** — snapshot before `MealRepository.delete`; delete throws if meal already removed.
3. **Edit persistence** — in-place update; preserve `id`/`userId`/`timestamp`; replace `FoodItem` children; HK reverse old + log new.
4. **Fiber target** — `(dailyCalorieTarget / 1000) × fiberGramsPer1000Kcal`; color at 90% / 70%. Fiber appears in **`MacroBarCard`** (progress bar) and **`DailySummaryFooterView`** (compact `Xg / Yg` text per spec).
5. **Dashboard refresh** — explicit `reloadDashboard()` on path pop / delete; `mealDetailReloadToken` refreshes detail after edit.
6. **Navigation** — `DashboardRoute` path with lightweight `UUID` cases (not `MealEntry` values).
7. **List layout** — non-scrolling `List` per section inside `ScrollView` (`MealListSectionView`).
8. **Empty meal sections** — all four `MealType` sections always visible.
9. **Delete confirmation** — alert for swipe and context-menu delete.
10. **Unsaved edit guard** — custom back button + `InteractivePopGestureDisabler` when scanner has unsaved work.
