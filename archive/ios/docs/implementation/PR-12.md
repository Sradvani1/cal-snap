## 8. App Store privacy & operator notes

> **Not** `PrivacyInfo.xcprivacy` contents — this section is for **App Store Connect** and Review Notes.  
> **Default rule used in PR12:** manifest/operator docs match runtime; code changed only when behavior is wrong (no runtime changes in this PR).

### Privacy Nutrition Labels (App Store Connect)

| Data type | Collected | Linked to identity | Tracking | Purpose | Operator notes |
|-----------|-----------|-------------------|----------|---------|----------------|
| Health & Fitness | Yes | No | No | App functionality | **Read:** body weight only (`fetchLatestWeight`), optional via Settings toggle. **Write:** dietary energy, protein, carbs, fat, fiber, body mass when writes enabled. Height authorized but **not read** (W-05). |
| Photos/Videos | Yes | No | No | App functionality | Camera + photo library for meal scan; JPEG stored locally in SwiftData `photoData`. |
| User Content | Yes | No | No | App functionality | Meal logs, profile fields, weigh-ins — local SwiftData only. |

**Do not select** “Data Not Collected.”

### Third-party / off-device processing

| Service | When | Data sent | Operator action |
|---------|------|-----------|-----------------|
| **Google Gemini API** | User taps analyze / tests API key | Meal image (JPEG) + optional text description; user-supplied API key from Keychain | Review Note: *“Meal scanning requires a user-provided Gemini API key. Images/descriptions are sent to Google’s Generative Language API over HTTPS. No CalSnap server.”* |
| **USDA FDC API** | — | **Not called in v1** — optional API key UI only; demo key constant unused at runtime | No separate label required; mention in privacy policy if published |

### Capability-specific notes

| Capability | User trigger | Runtime behavior | Connect / plist |
|------------|--------------|------------------|-----------------|
| **HealthKit** | Onboarding + Settings toggles | Read weight; write meals + weigh-in; fire-and-forget writes | Entitlement + share/update usage strings in Info.plist |
| **Camera / Photos** | Scanner capture / picker | Local storage; optional Gemini upload on scan | `NSCameraUsageDescription`, `NSPhotoLibraryUsageDescription` |
| **Notifications** | Weigh-in schedule; optional daily log (default **off**) | Local `UNUserNotificationCenter` only | `NSUserNotificationsUsageDescription` (weekly + optional daily) |
| **Widget** | User adds widget | Read-only `WidgetData` from App Group; no network/HK in extension | No extra collected-data type |
| **Export CSV** | Settings → Export | User-initiated temp file via share sheet | User-controlled export |
| **Delete all data** | Settings destructive action | SwiftData wipe + notification cancel + `WidgetSyncService.clear()` | Document in privacy policy if published |
